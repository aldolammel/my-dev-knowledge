

DEV CONCEPTS: VIRTUAL PRIVATE SERVER (VPS)

    Contrary to shared hosting, which is already configured with many apps and ready to serve your app to the internet, a VPS comes with only the OS. It's your responsibility to configure it to serve your app.
        
    Some VPS you need to get in the Control Panel in the provider website to select the OS to be installed automatically.
    
    USE STRATEGY:
        
        >> Dedicated Instance:
            - Each VPS runs a single application.
            - Pros: full resource isolation, independent scaling, simpler rollbacks, no "noisy neighbor" problem.
            - Cons: expensive, wasteful if apps are small, more infra to manage.
        
        >> Consolidated Server:
            - Multiple apps coexist on a single VPS, separated by ports/domains via a reverse proxy (Nginx/Caddy).
            - Pros: cost-efficient, simpler for small portfolios.
            - Cons: resource contention, harder isolation, a crashing process can affect others.
            
        >> Docker / Podman:
            - Apps run in containers on one or more VPS. Isolation is at the container level, not the machine level.
            - Pros: near-full isolation, reproducible environments, easier deploys, scales toward orchestration.
            - Cons: Docker overhead, complexity for small teams.
