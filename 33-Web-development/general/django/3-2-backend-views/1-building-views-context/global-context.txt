

    GLOBAL CONTEXT:

        >> If you need a Context that need to be available on the entire application,
            you want a 'Global Context'!


        0) Where is the data you want to make global? For example purposes, let's use
            custom data defined in settings.py!


        1) In a sub-app (recommend the 'general' one), create a file called 'context_processors.py':

            from django.conf import settings as stt
            from django.utils import timezone


            def app_info(request):
                """
                Information about the app:
                """
                return {
                    'BRAND_NAME': stt.BRAND_NAME,
                    'BRAND_EMAIL': stt.BRAND_EMAIL,
                    'BRAND_URL': stt.BRAND_URL,
                    'DEV_NAME': stt.DEV_NAME,
                    'DEV_URL': stt.DEV_URL,
                    'ADMIN_EMAIL': stt.ADMIN_EMAIL,
                    'CURRENT_YEAR': timezone.now().year,
                }

        
        2) In settings.py file, find 'TEMPLATES' section and add:

            TEMPLATES = [
                {
                    ...
                    'OPTIONS': {
                        'context_processors': [
                            # DJANGO DEFAULT GLOBAL CONTEXTS:
                            ...
                            # DJANGO ADDITIONAL GLOBAL CONTEXTS:
                            ...
                            # THIRD-PARTY GLOBAL CONTEXTS:
                            ...
                            # APP CUSTOM GLOBAL CONTEXTS:
                            'general.context_processors.app_info',
                        ],
                    },
                },
            ]


        3) Now, call your global context wherever you want on templates:

            E.g.

                <a href="mailto:{{ BRAND_EMAIL }}?subject={{ BRAND_NAME }}: Contact from website" target="_blank">Contact us</a>
