

    GLOBAL CONTEXT:

        >> If you need a Context that need to be available, for example, in the entire application,
            you want a 'Global Context', e.g. Dropbox with Interface Language Options!


        1) In a sub-app (recommend the 'general' one), create a file called 'context_processors.py':

            from django.conf import settings

            def your_global_feature_example(request):
                return {
                    'LANGUAGES': settings.LANGUAGES,  # Just context content examples!
                    'LANGUAGE_CODE': request.LANGUAGE_CODE,
                }

        
        2) In settings.py file, find 'TEMPLATES' section and add it:

            TEMPLATES = [
                {
                    ...
                    'OPTIONS': {
                        'context_processors': [
                            # DJANGO DEFAULT GLOBAL CONTEXTS:
                            ...
                            # DJANGO ADDITIONAL GLOBAL CONTEXTS:
                            'general.context_processors.your_global_feature_example',
                            # THIRD-PARTY GLOBAL CONTEXTS:
                            ...
                            # PRODUCT CUSTOM GLOBAL CONTEXTS:
                            ...
                        ],
                    },
                },
            ]


        3) Now, call your global context wherever you want on templates:

            E.g.

                <select name="...">
                    {% for lang_code, lang_name in LANGUAGES %}
                        <option value="{{ lang_code }}"{% if lang_code == LANGUAGE_CODE %} selected{% endif %}>
                            {{ lang_name }}
                        </option>
                    {% endfor %}
                </select>
