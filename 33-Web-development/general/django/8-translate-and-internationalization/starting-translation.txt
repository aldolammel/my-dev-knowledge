

    STARTING A TRANSLATION PROJECT:


        0) Install the gettext module:

            $ sudo apt install gettext
        
        
        1) In settings.py, add this Middleware between the 'CommonMiddleware'
            and 'CsrfViewMiddleware':
        
                'django.middleware.locale.LocaleMiddleware',
        
        
        2) Still in settings.py:

            # Internationalization
            # https://docs.djangoproject.com/en/5.0/topics/i18n/
            USE_I18N = True
            LANGUAGES = [
                ('en-us', 'English'),
                ('pt-br', 'Brazilian Portuguese'),
            ]
            LANGUAGE_CODE = 'en-us'  # Default language
            LOCALE_PATHS = [BASE_DIR / 'locale']
            # Doc: http://en.wikipedia.org/wiki/List_of_tz_database_time_zones
            USE_TZ = True
            TIME_ZONE = 'America/Sao_Paulo'  # 'UTC'
            # Date and time format settings
            DATE_FORMAT = 'Y-m-d'
            TIME_FORMAT = 'H:i:s'
            DATETIME_FORMAT = 'Y-m-d H:i:s'
            SHORT_DATE_FORMAT = 'd/m/Y'
            SHORT_DATETIME_FORMAT = 'd/m/Y P'
            FIRST_DAY_OF_WEEK = 1  # 0 = Sunday
            # Input format settings
            DATE_INPUT_FORMATS = [
                '%d-%m-%Y',
                '%d/%m/%Y',
                '%d%m%Y',
            ]
            TIME_INPUT_FORMATS = [
                '%H:%M',
                '%H%M',
            ]
            DATETIME_INPUT_FORMATS = [
                '%Y-%m-%d %H:%M',
                '%Y/%m/%d %H:%M',
            ]



        X) Still in the config-folder, edit the main urls.py file:

            from django.conf.urls.i18n import i18n_patterns

            urlpatterns = i18n_patterns(
                <your paths here with no list/array!>
            )


            >> Tweaks right above make Django to identify the respective language-code has been used:
                http://127.0.0.1:8000/en/
                http://127.0.0.1:8000/pt-br/
                ...



        3) Get started to translate the content through these methods for Views and Models:

            from django.utils.translation import gettext_lazy as _  # This '_' is a convention!

            class Product(models.Model):
                name = models.CharField(
                    verbose_name=_("Product Name"),
                    ...
                )
                description = models.TextField(
                    verbose_name=_("Description"),
                    ...
                )


        4) In order to translate templates:

            4.1) Call this in the FIRST line on each html file that's using
                translation benefits, EXCEPT if the template is calling a 'extends':
                    
                    {% extends "base.html" %}

                    {% load i18n %}

                >> Crucial: the 'load' must be called even in 'includes'.


            4.2) Translating the HTML content itself:
                
                    <h1>{% trans "Welcome" %}</h1>
                    <p>{% trans "I am Aldo Lammel" %}.</p>
        

        5) Create the translate folder and files:

            5.1) Create the 'locale' folder in the project root;

            5.2) Create each additional language file. This will create a .po file for
                the pt_BR locale under the /locale/ path. E.g.:

                    $ django-admin makemessages -l pt_BR

                    or

                    $ django-admin makemessages --all


        6) Translating proccess:

            msgid "Welcome"
            msgstr "Bem-vindo(a)"

            msgid "I am Aldo Lammel"
            msgstr "Me chamo Aldo Lammel"


        7) Compiling translations to Machine Object (.mo): once you've translated the strings in
            the .po file, you need to compile them for Django to use:

                $ django-admin compilemessages


        8) Language switching (template):

            <form action="{% url 'set_language' %}" method="post">
                {% csrf_token %}
                <input name="next" type="hidden" value="{{ redirect_to }}">
                <select name="language">
                    {% for lang_code, lang_name in LANGUAGES %}
                        <option value="{{ lang_code }}"{% if lang_code == LANGUAGE_CODE %} selected{% endif %}>
                            {{ lang_name }}
                        </option>
                    {% endfor %}
                </select>
                <input type="submit" value="Change Language">
            </form>


        9) Database: If your database contains any content that will be translated,
            ensure your PostgreSQL database is set up with UTF-8 encoding, which is necessary
            for handling multi-language content.

                $ psql -U yourusername -d yourdbname -c
