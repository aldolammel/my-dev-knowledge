DJANGO TEMPLATES: BASIC >> Hashtag signs with curly-braces {# #} are TEMPLATE-COMMENTS that will be only server-side, unlike when is used the . E.g. {# This is just a server-side comment! #} >> Percentage signs within curly-braces {% %} are TEMPLATE-TAGS used to perform logic operations, control structures, and include other template functionalities within a template. E.g. >> Calling another template as base for this template: {% extends 'base.html' %} >> The 'base.html' example you find here: /33-Web-development/general/django/project-examples/proj-common/templates/base.html >> Defining the template's content-block must be loaded here: {% block content %} {% endblock %} >> A conditional example: {% if cars %}

Good, there is at least one car available!

{% else %}

Oops, there is no car available!

{% endif %} >> A For loop example: {% for car in cars %} {% comment %} Applying dynamic links/routes namespace and pattern-name are defined in each sub-app urls.py file {% endcomment %}

xxxxxxxx

{% endfor %} >> Double curly-braces {{ }} are TEMPLATE-VARIABLES which are used to output the value of a variable sent by the CONTEXT (dictionary) from a views.py file.
