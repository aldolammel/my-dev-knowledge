

DJANGO > PROJECT INSTALLATION: USING UV AS PACKAGE MANAGER

    PRE.1) If you wanna re-install an existing project, skip this roadmap, and go to:
        ./1-re-install-project-with-uv.txt

    PRE.2) Assuming you already have the UV package manager installed:
        /Python/Package-manager/uv/_about-install-and-update.txt


    - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -


    >> BEFORE PROJECT INSTALLATION:
        
        PRE.1) DON'T create venv yet!

        PRE.2) Make sure you already installed all Python versions for this project:

            >> Check which Python versions you got installed:
                $ uv python list
            
            >> E.g. you want to install more than one Python version:
                $ uv python install 3.11 3.12 3.13


    - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -


    >> PROJECT CREATION:

        1) Local project folder:
            
            >> Don't create any additional folders beyond the project folder you already created, but mentally define which Django folder structure you will use further:
                .../django/django-project-folder-structure.txt
        
        
        2) Assuming you already opened the Project folder through an IDE;


        3) (If applicable)
            Project versioning:
                >> Create a repository or use an existing one:
                    /Versioning/git/command-init.txt
        
        
        4) Creating new project:
                
            4.1) Create the virtual environment folder:
                /Python/Package-manager/uv/create-or-find-current-venv.txt

                >> Do you need a pyproject.toml model?
                    /Python/Web-development/pyproject.toml

            4.2) (NOT SURE IF IT'S NEEDED AFTER 4.1)
                Create the minimum Python project scaffolding:
                    
                ATTENTION:
                    If you wanna use existing pyproject.toml and .gitignore files for example, be aware once the 'init' command will create those files with others, overridden files in project folder if not empty!

                # That said, do it:
                    $ uv init
            
            4.3) Once you now got a .venv folder, active the project's virtual environment:
                /Python/3-virtual-environment/activate-and-deactivate.txt


            4.4) Other stuff:

                >> Django basics:
                    .../django/IDE-softwares/vscode/examples/settings.json
                    .../django/IDE-softwares/vscode/examples/extensions.json

                >> Check if the Python version on that file is correct!
                    If NOT okay, fix it!
                        E.g.
                            requires-python = ">=3.13,<3.14"
                                
                >> Check the .python-version file:
                
                    >> In case this file has the wrong Python version or versions, don't edit directly this file. Use the pin command:
                                
                        /Python/Package-manager/uv/pin-python-version.txt


            4.5) Install Django in this project:
                
                # Install a specific version (recommended):
                    $ uv add django==5.2.7
                # Or the latest version:
                    $ uv add django


            4.6) Define your database case it SHOULDN'T use SQLite:
                .../django/3-1-models-database/0-installing-and-adminUser/_define-the-database.txt


            4.7) After Django installation:

                4.7.1) Create the Django Project structure, going to the project root folder and asking the package manager to create the 'core' folder
                
                    $ uv run django-admin startproject core .              <-- This dot's important!

                4.7.2) (If applicable)
                    Case you need an admin user:
                        .../django/3-1-models-database/0-installing-and-adminUser/creating-admin-user.txt


            - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - 

        5) (Optional)
            Based on you already know or in the Engineering Data document of the project, install all project dependencies and its development dependencies too:
                
                /Python/Package-manager/uv/install-dependency.txt

                # The Engineering Data document link must be registered in the Django core settings.py file!
        
        
        6) (Optional)
            If you have multilingual support in your project:

            >> With Gettext module installed:
                
                $ uv run manage.py makemessages --all
                $ uv run manage.py compilemessages

            
        7) (If applicable)
            Make sure you got your .gitignore updated with Django, environment and other things:
                /Versioning/git/gitignore-file.txt


        8) Check Django installation and setup:
            ./2.1-installed-project-testing.txt



    - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - 
        Since this point, you might execute basic tests without the follow steps once your goal
                            would be to test something basic on Django.
    - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - 



        9) (If applicable)
            Commit files in your versioning service (e.g. GitHub)!


        10) Let's setup the new project:
            ./2.2-installed-project-setup.txt
