

DEBIAN/UBUNTU > BUILDING UP A VPS: DEFINING ROOT USER AND OS


    PRE.1) Warnings!
    
        VPS CONCEPT & STRATEGIES:
            /vault/dev-concepts/server-vps.txt
    
        CRUCIAL:
            Always check if your VPS setup is allowed to run healthily the chosen OS!
        
        ROOT PASSWORD:
            Don't play with your VPS root user. Use a strong pass always, and save it in a project doc! This Norton tool helps you to generate a strong pass:
                https://us.norton.com/feature/password-generator         
        
        
    PRE.2) VPS reinstallation?
        ./dedicated-instance/_reinstalling-a-server.txt
    

    1) Connect to the server:
        Usually, you receive an email with the credentials required to connect it by terminal. Open your local terminal and:
    
            /vault/server/connecting-via-ssh.txt
            
            INFO
                In the very first time, say 'yes' to fingerprint creation that will address this new server in your user 'known_hosts' file!
            
            # Public Key error?
                If you have no other servers to worry about, make a copy of your user 'known_hosts' ('known_hosts.old') and then delete the original file:
                    
                    >> Debian/Ubuntu OS:
                        /root/.ssd/known_hosts
                        
                        
    2) Double-check: make sure what OS the VPS got installed:
        $ lsb_release -a
        

    3) Update the server to its last components and dependencies versions:
        
        3.1) Update the repos and install the updates available:
            
            >> Debian/Ubuntu:
                /vault/os/os/linux/debian/2-updates/
            
            >> Fedora:
                /vault/os/os/linux/fedora/2-updates/
        
        3.2) After that, do you wanna change the OS version?
            Don't OS release upgrade (LTS or non-LTS version) without to check if the current VPS setup supports the new version!
                
            >> Debian/Ubuntu:
                /vault/os/os/linux/debian/2-updates/changing-LTS-version.md
                
            >> Fedora:
                /vault/os/os/linux/fedora/2-updates/changing-LTS-version.md
        
    
    4) Restart the VPS:
        $ reboot
