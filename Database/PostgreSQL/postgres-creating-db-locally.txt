

POSTGRESQL: CREATING A DATABASE LOCALLY

    PRE.1) Assuming you got already the PostgreSQL installed locally:
        ./0-installing-and-updating/postgres-installing.txt

    PRE.2) Make sure the database is running:
        ./postgres-starting-and-stoping.txt

    >> Method to create a database:

        A) By pgAdmin;
        B) By terminal;

        - - - - - -

        A) Using PgAdmin - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -

            A1) Open your local PgAdmin;
            
            A2) In the server tree (sidebar-left), also open the 'Databases';
            
                ATTENTION:
                    Never use the database called "postgres". This is the default db and it cannot be delete!

            A3) Create the new database:
                >> Right-click on "Databases" and select "Create" > "Database...";
                >> In the "Database" field, enter the name:
                        
                    By convention:
                        
                        <Cliente>_<Product_type>_<Release_year>
                    
                        e.g.:
                            Abcoo_b2b_2030
                            Abican_portal_2023
                            Aldolammel_ecommerce_2020
                            Ammana_b2b_2019

                >> And then save it! If no error, it's done, otherwise:

                    ATTENTION:
                    If an error message says "Collation version mismatch" it's because your O.S. has been updated but PostgreSQL db cluster hasn't! Solution:

                        A3.1) Close the PgAdmin, and stop the Postgres service:
                            /Database/PostgreSQL/postgres-starting-and-stoping.txt

                        A3.2) Update and upgrade your Postgre and O.S;

                            UBUNTU:
                                A3.2.1) Add the Postgre repo locally:
                                    ./0-installing-and-updating/postgres-updating.txt

                                A3.2.2) Update the entire system:
                                    /OS/debian/2-updates/update.txt

                            WINDOWS:
                                A3.2.1) xxxxxxxxxxxxxxx

                                A3.2.2) xxxxxxxxxxxxxxx
                        
                        A3.3) Start the Postgres again:
                            ./postgres-starting-and-stoping.txt

                        A3.4) Update the collation version:
                            ./postgres-collation-updating.txt

                        A3.5) Open again the PgAdmin, and now you should be able to create a new db!



        B) Using Terminal - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - 

            PRE) Update the collation version:
                ./postgres-collation-updating.txt

            B.1) Define with encoding for your project db (root user = postgres):

                F) English language based;
                G) Or Brazilian Portuguese language based;

                - - - - 

                F) English language based:
                    $ CREATE DATABASE your_project_db_name OWNER your_django_db_user ENCODING UTF8 LC_COLLATE 'en_US.UTF-8' LC_CTYPE 'en_US.UTF-8' TEMPLATE template0;

                G) Brazilian Portuguese language based:
                    $ CREATE DATABASE your_project_db_name OWNER your_django_db_user ENCODING UTF8 LC_COLLATE 'pt_BR.UTF-8' LC_CTYPE 'pt_BR.UTF-8' TEMPLATE template0;

                >> Facing collation error:
                    ./postgres-collation-updating.txt

            - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - 

            B.2) Exit Postgres shell:
                $ \q
