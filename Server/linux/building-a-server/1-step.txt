

DEBIAN/UBUNTU > BUILDING UP A WEB-SERVER: DEFINING THE SERVER OS


    PRE) Install the server OS:
    
        Contrary to shared hosting, which is already configured with many apps and ready to serve your app to the Internet, a VPS comes with only the operating system. It is your responsibility to configure it to serve your app.
        
        Some VPS you need to get in the control panel in the provider website to select the OS to be installed automatically by the provider. 
        
        So, do it if needed <-----


    1) Connect to the server:
        Usually, you receive an email with the credentials required to connect it by terminal. Open your local terminal and:
    
            /Server/connecting-via-ssh.txt
            
            INFO
                In the very first time, say yes whatever is shown, and congrats, you're in!
            
            Public Key error?
                If you have no other servers to worry about deleting the "known_hosts" file, delete it:
                    Windows OS:
                        <WINDOWS_DRIVE>:\Users\<your_user>\.ssh\known_hosts
                    Debian/Ubuntu OS:
                        xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
                        
                        
    2) Double-check: make sure what OS the VPS got installed
        $ lsb_release -a
        

    3) Update the server to its last version as well as its components:
        
        3.1) Do it:
            
            >> Debian/Ubuntu:
                /OS/linux/debian/2-updates/
            
            >> Fedora:
                /OS/linux/fedora/2-updates/
        
        3.2) After that, do you wanna change the OS version?
            Don't OS release upgrade (LTS or non-LTS version) without to check if the current VPS setup supports the new version!
                
            >> Debian/Ubuntu:
                /OS/linux/debian/2-updates/changing-LTS-version.txt
                
            >> Fedora:
                /OS/linux/fedora/2-updates/changing-LTS-version.txt
        
    
    4) Restart the VPS:
        $ reboot
