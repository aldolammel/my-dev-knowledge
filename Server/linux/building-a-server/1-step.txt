

DEBIAN/UBUNTU > BUILDING UP A SERVER: DEFINING THE SERVER OS


    PRE.1) What is VPS (server):
    
        Contrary to shared hosting, which is already configured with many apps and ready to serve your app to the internet, a VPS comes with only the operating system. It's your responsibility to configure it to serve your app.
        
        Some VPS you need to get in the control panel in the provider website to select the OS to be installed automatically by the provider. 
        
        PRE.1.1) So, do it if needed, checking if your VPS setup is allowed to run healthily the chosen OS!
        
        PRE.1.2) Define the root user's password. 
            TIP: Use the Norton Generated Password for it and save it in the project documentation!
                https://us.norton.com/feature/password-generator         
        
        
    PRE.2) For VPS reinstallation, make sure you have read the initial reinstall steps:
        ./_reinstalling-a-server.txt
    

    1) Connect to the server:
        Usually, you receive an email with the credentials required to connect it by terminal. Open your local terminal and:
    
            /Server/connecting-via-ssh.txt
            
            INFO
                In the very first time, say 'yes' to fingerprint creation that will address this new server in your user 'known_hosts' file!
            
            # Public Key error?
                If you have no other servers to worry about, make a copy of your user 'known_hosts' ('known_hosts.old') and then delete the original file:
                    
                    >> Windows OS:
                        <WINDOWS_DRIVE>:\Users\<your_user>\.ssh\known_hosts
                    
                    >> Debian/Ubuntu OS:
                        /root/.ssd/known_hosts
                        
                        
    2) Double-check: make sure what OS the VPS got installed:
        $ lsb_release -a
        

    3) Update the server to its last version as well as its components:
        
        3.1) Update the repos and install the updates available:
            
            >> Debian/Ubuntu:
                /OS/linux/debian/2-updates/
            
            >> Fedora:
                /OS/linux/fedora/2-updates/
        
        3.2) After that, do you wanna change the OS version?
            Don't OS release upgrade (LTS or non-LTS version) without to check if the current VPS setup supports the new version!
                
            >> Debian/Ubuntu:
                /OS/linux/debian/2-updates/changing-LTS-version.txt
                
            >> Fedora:
                /OS/linux/fedora/2-updates/changing-LTS-version.txt
        
    
    4) Restart the VPS:
        $ reboot
