

DEBIAN/UBUNTU > BUILDING UP A WEB-SERVER: DEPLOYING A WEB APP


    PRE.1) Assuming you have finished the previous step!
    
    
    PRE.2) You're in the VPS terminal!
    
    
    1) Once the VPS is to store a project's product, create the project folder in the convention folder, not attached to any user specifically:
    
        # Where you are:
            $ pwd
            
            # To list the folder's content:
                $ ls
                
        1.1) Go to: 
            /var/www/
        
        1.2) Create the project folder:
            $ sudo mkdir <project_folder>
                E.g.
                    portfolio_2030
        
        KEEP IT IN MIND:
            1 - Project path: /var/www/<project_folder>
            2 - DON'T CREATE VENV FOLDER YET ONCE GIT CLONE WOULD CONFLICT WITH IT!
            

    2) Install Python:
    
        Before you install Python on Linux VPS, you should check whether it's already installed by default (probably yes):
            /Python/1-python-installation/_install.txt
        
        If you want to update the Python version, it's advised to remove the old one to avoid two or more Python version running:
            /Python/1-python-installation/uninstall.txt


    3) Python basic modules:
    
        3.1) Install a package manager (Don't create VENV yet!):
    
            >> PIP:
                Built-in Python solution, but the Python in Debian doesn't bundled PIP. So do it manually if needed!
                
                /Python/Package-manager/pip/
            
            >> UV:
                /Python/Package-manager/uv/
                    # In this case, restart your terminal shell!
                    
        3.2) Install Python package dependency for NOT PURE Python modules:
            $ sudo apt install -y python3-dev
        
        3.3) (If applicable) Only for non-UV users:
            
            >> Install the python-venv module (to create, further, the venv):
                $ sudo apt install -y python3-venv
            
       
    4) Installing a database:
    
        >> For SQLite3 (Python bundled db solution):
            /Database/SQLite/0-basic/1-installing-and-integrating.txt

    
    5) Installing a web application (based in an existing repository):
        
        5.PRE.1) Assuming you are in VPS project folder yet!
        
        5.PRE.2) Git is installed on the server:
            /Versioning/git/_basic.txt
        
        5.1) Clone your project repo from GitHub here:
            /Versioning/git/command-clone.txt
            
            CRUCIAL:
                Use "SSH" hyperlink version on GitHub to clone the repo. That will ask for Public Key already created!
            
        5.2) GitHub will ask for the Public Key password (that I advice you to create w/ 20+ chars)!
            
        5.3) Create a venv folder in the VPS project folder:
    
            5.3.1) Creation: 
                >> Using Python-venv module:
                    /Python/3-virtual-environment/creating-virtual-environment.txt
                    
                >> Using UV:
                    /Python/Package-manager/uv/create-or-find-current-venv.txt
                    
            5.3.2) Active the venv:
                /Python/3-virtual-environment/activate-and-deactivate.txt
                
        5.4) Installing project dependencies:
        
            >> PIP users:
                /Python/Package-manager/pip/auto-installation-from-requirements-file.txt
            
            >> Or UV users:
                /Python/Package-manager/uv/auto-installation-with-sync.txt
    
    
    6) Building the bridge between the Webserver and the app's code language:
        
        6.1) Which language you are working:
        
            >> For Python projects:
            
                >> Gunicorn solution:
                    /Server/_languages-server-tools/python/gunicorn/install.txt
                    
        6.2) (If applicable) Update the firewall:
            
            >> Using UFW:
                $ sudo ufw allow <language_port>
            
            >> Which language:
                >> For Python projects, port = 5000;          
                
            >> Check firewall current rules:
                >> Using UFW:
                    $ ufw status verbose
        
                    
    x) Environment variables on Production (VPS):
    
        x.1) (If applicable) Once your project is on prod, set the VPS environ vars:
            
            # Open the Env Vars list of the VPS:
                $ sudo nano /etc/environment

            # Copy the env vars from the back-end project .env file to the VPS!
                
                ATTENTION:
                    Check if all env var values make sense to go like that to prod!
                
                CRITICAL:
                    Don't be stupid as hell to send your .env file to the repo, or even to the prod!
                
            # Save it (ctrl+s) and exit (ctrl+x).
            
            # Logout and reconnect to the VPS to check the new env vars:
                $ exit
                $ <connect again>
                $ printenv
  
    
    x) Testing the application:
    
        x.1) Preparing a testing file:
        
            # Replace your product main file for one of them below just for test if the server is printing out the product:
        
            >> For Python > Flask projects:
                /Python/Web-development/flask/z_examples/_minimal-code.py
            
            >> For Python > Django projects:
                /xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
                
        x.2) Check if the production environment is showing the project product on browser:
        
            x.2.PRE.1) Assuming the prod environment is using a 'hello world' version for testing!
            
            x.2.PRE.2) Assuming you're in VPS with the right venv activated!
            
            x.2.1) Run the test:
                # For Python:
                    $ python3 <project_main_file>.py

            x.2.2) Check on browser:
                >> For Python:
                    http://<vps_address>:5000
                    
                # The port is already in use?
                    $ ps -fA | grep python
                    
                    Output e.g.
                        root   10445   10351   0   19:20   pts/1   00:00:00 /var/www/xxxx/venv/bin/python /var/www/xxxx/venv/bin/flask run
                        The process id (pid) is the second number (10351). So if you want to use the default port (5000), stop the process that is using it:
                    
                    $ kill 10351
                        # And then try again!
