

DEBIAN/UBUNTU > VPS: UPDATING A WEB APPLICATION


    PRE) Keep in mind: - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
        
        Current application files:
            Keep your application GIT repo accessible on the browser. e.g.:
                https://github.com/xxxxxxxxxx/tree/main

        Current application main domain:
            E.g.
                http://vpsXXXXX.publiccloud.com
        
        Current app path on server:
            E.g.
                $ cd /home/admin/portfolio
                
        Updating database:
            For example, I'm currently using the portfolio repo to store the DB ("instance" folder).


    1) Updating files: - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - - -
        
        1.1) Connect to the server:
            $ ssh -p 4928 root@<server-ip>

        1.2) Update the application repository:
            $ sudo git pull

        1.3) Refresh the communication between the web server and the app, restarting the Gunicorn (WSGI):
            $ systemctl restart <your_application_name>   # e.g. portfolio
            $ systemctl status <your_application_name>

    2) Something wrong? Try it:
        # Restart the web server:
        $ systemctl restart nginx
        $ systemctl status nginx

        # Restart the communication between the web server and the web application:
        $ systemctl restart <your_application_name>
        $ systemctl status <your_application_name>

        # Nothing yet? Restart the server and repeat this entire step:
        $ reboot
